# Практическая работа №1: Реализация серверной конфигурации с Docker и Docker Compose

Данный проект демонстрирует реализацию серверной конфигурации в соответствии с требованиями "Практической работы №1.pdf", используя Docker и Docker Compose для оркестрации сервисов.

## Выполненные задачи:

### 1. Конфигурация веб-сервера с PHP (`Dockerfile`)

Файл `Dockerfile` определяет образ для веб-сервиса:
*   **Базовый образ:** Используется `php:7.4-apache`, предоставляющий окружение PHP версии 7.4 с предустановленным веб-сервером Apache. Это соответствует требованию наличия PHP и веб-сервера.
*   **Установка расширений:** Устанавливается и активируется расширение `mysqli`, необходимое для взаимодействия PHP с базой данных MySQL.
*   **Открытие порта:** Порт `80` (стандартный для HTTP) открывается в контейнере, делая веб-сервер доступным.

### 2. Оркестрация сервисов (`docker-compose.yml`)

Файл `docker-compose.yml` организует два основных сервиса: `web` (веб-сервер) и `db` (база данных MySQL).

#### Сервис `web`:
*   **Сборка образа:** Инструкция `build: .` указывает на сборку образа из текущей директории, используя предоставленный `Dockerfile`.
*   **Маппинг портов:** Порт `8080` хост-машины маппируется на порт `80` внутри контейнера, что позволяет получить доступ к веб-приложению через `http://localhost:8080`.
*   **Монтирование тома:** Текущая директория проекта монтируется в `/var/www/html` контейнера. Это позволяет `index.php` и другим файлам проекта быть доступными для Apache внутри контейнера.
*   **Зависимость:** `depends_on: - db` гарантирует, что сервис базы данных будет запущен до старта веб-сервиса.

#### Сервис `db`:
*   **Образ базы данных:** Используется официальный образ `mysql:5.7`, что соответствует требованию наличия СУБД MySQL.
*   **Переменные окружения:** Установлены ключевые переменные окружения для конфигурации MySQL:
    *   `MYSQL_ROOT_PASSWORD: root_password`
    *   `MYSQL_DATABASE: appDB`
    *   `MYSQL_USER: user`
    *   `MYSQL_PASSWORD: password`
    Эти параметры соответствуют настройкам подключения, используемым в `index.php`.
*   **Постоянное хранение данных:** Том `db_data` используется для сохранения данных MySQL, обеспечивая их персистентность.
*   **Инициализация базы данных:** Файл `./init.sql` монтируется в `/docker-entrypoint-initdb.d/init.sql` внутри контейнера MySQL. Это критически важно для выполнения задания, так как `init.sql` будет автоматически выполнен при первом запуске контейнера, создавая необходимую базу данных, пользователя и таблицу `users` со столбцами `ID`, `name` и `surname`, как того требует задание.
